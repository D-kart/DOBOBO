现状盘点

我们已经手握三块金子
但还缺让它"活起来"的底盘

✓ 已具备 · 三块金子
  • 四表合一内容资产——备孕 28 天、孕期 280 天、0~6 岁分龄到周/天的执行卡，全网稀缺。
  • 东方理念护城河——矢能导育、新六艺、传家故事，与抢跑式早教完全错位的叙事。
  • 线下闭环已跑通——工作坊 + 阿舅 + 普惠定价（12800/年），现金流和口碑模型清晰。
✗ 缺什么 · 五处缺口
  • 没有"本体"——内容散在 docx / md / 群聊里，没统一标签、没分龄维度索引。
  • 没有"数据契约"——共建格式不统一，没法机器消费、没法进模型。
  • 没有"治理机制"——谁能写、怎么审、版本怎么走、贡献怎么记，没成文。
  • 没有"采集回流"——小程序、阿舅、工作坊数据没进同一个池子。
  • 没有"评测与红线"——AI 答错、医学误导、伦理边界，目前完全靠人工把关。
结论

我们不缺内容，缺的是把内容变成"养育垂类智能体可消费的资产"的那一层底座。
3 个月的力气全押在这一层，1 年后没人追得上。

TECH · DATA CONTRACT

一张内容卡的字段解剖

JSON Schema 强校验 · 字段不全不入库 · 版本永久可回溯

practice_card.schema.json · v1.0 
// DOBOBO Practice Card · 强校验 Schema
{
  "$schema": "https://json-schema.org/draft-07/#",
  "type": "object",
  "required": ["id","title","age_range_d",
              "action","source_canon","scene_signal"],
  "properties": {
    "id":           { "type": "string", "pattern": "^DBB-[A-Z0-9]{8}$" },
    "title":        { "type": "string", "maxLength": 40 },
    "phase":        { "enum": ["备孕","孕期","0-1Y","1-3Y","3-6Y"] },
    "age_range_d":  { "type": "array", "items":{"type":"integer"}, "minItems":2 },
    "scene_signal": {                              // ★ 主动智能关键字段
      "type": "object",
      "properties": {
        "signal_type": { "enum": ["time","behavior","voice","vision"] },
        "conditions":  { "type": "array" },
        "cooldown_h":  { "type": "integer", "default": 24 }
      }
    },
    "action":       { "type": "string", "maxLength": 240 },
    "warning":      { "type": "string" },
    "source_canon": {                              // 东方典籍引文
      "type": "array",
      "items": { "required":["book","chapter"] }
    },
    "evidence_modern":{                            // 现代科学证据
      "type": "array",
      "items": { "properties":{"doi":{"type":"string"}} }
    },
    "confidence":   { "type": "number", "minimum":0, "maximum":1 },
    "contributor":  { "type": "string" },
    "version":      { "type": "string", "pattern": "^v\\d+\\.\\d+$" },
    "redline_passed":{ "type": "boolean" }
  }
}
idstring · regex
全局唯一卡片 ID，DBB-XXXXXXXX 8 位大写字母数字，可寻址、可回滚、可被外部引用。
age_range_dint[2]
分龄精度到天。比如 [1,28]=备孕 28 天，[280,365]=出生后 0~85 天。
scene_signalobject · ★ 关键
软硬一体的接口。眼镜识别"宝宝在哭"=voice 信号、导育机识别"已 2 周未茶时刻"=behavior 信号——主动触达的训练原料。
source_canon
+ evidence_modernarray
双轨证据链。东方典籍引文 + 现代科研 DOI 同时挂载，模型作答必带出处。
confidencenumber 0~1
综合可信度评分 = f(来源权威度, 共建反馈, 红线通过)。低于 0.6 不入推理流。
version + contributorSemVer + git
版本号 + 贡献者署名永久挂——每改一次都留痕，可秒级回滚。
redline_passedboolean
三审一公示走完才置为 true，未通过的卡永远不会被推理服务调到。

用户侧产品一号位的提案

我建议下午就拍板的三件事

建 议 一

底座先行
App 先收口不先扩

把"四表合一"先用同一套数据结构装进库；小程序只做"按天发卡 + 打卡 + 反馈"这一条主链路，其他模块全部冻结到 6 月底。

为什么

功能多 ≠ 价值大。底座没立住，App 加再多功能都是技术债。M1 的克制，是 M3 的爆发力。

建 议 二

共建治理
立刻成文

一周内出《DOBOBO 维基共建白皮书 v1》：哪部分官方锁死、哪部分开放、版本怎么走、贡献者署名怎么挂、争议怎么裁。

为什么

没有治理，共建就是失控；失控的内容比没有内容更危险——尤其养育垂类，会出医学事故和家庭伦理事故。

建 议 三

从 Day 1
就为软硬一体留接口

所有内容卡多一个字段：scene_signal——这个任务在什么家庭场景下应该被主动触发。这是未来眼镜/导育机"主动发现并提醒"的训练原料。

为什么

这一步现在不留，3 个月后回头补，要把 2000+ 张卡片全部重做一遍——成本翻 10 倍。典型的"现在 1 块，未来 10 块"杠杆点。
